# Project Structure

## Top Level

- `README.md`
  User-facing English README.

- `README.id.md`
  User-facing Bahasa Indonesia README.

- `CONTRIBUTING.md`
  Contributor workflow and expectations.

- `docs/`
  Developer and maintenance documentation.

- `src/`
  React application source.

- `src-tauri/`
  Tauri/Rust application source and generated mobile projects.

- `powersync/`
  PowerSync schema and sync rules.

- `public/`
  Static assets, including the PowerSync WASM worker assets.

- `.github/workflows/`
  Desktop and mobile release workflows.

## Frontend Layout

- `src/main.tsx`
  React entry point. Creates the router and wraps the app in shared providers.

- `src/routes/`
  TanStack Router file routes:
  - `/`
  - `/auth/sign-in`
  - `/auth/sign-up`
  - `/auth/forgot-password`
  - `/setup/store`
  - `/stores/$storeSlug`

- `src/components/submodules/`
  Most store features live here as self-contained modules.

- `src/lib/`
  Shared app logic, including:
  - auth setup
  - organization/store gate logic
  - i18n
  - PowerSync setup
  - AI runtime helpers

- `src/main.css`
  Global Tailwind, HeroUI, Neon auth UI, and Streamdown styling.

## Rust and Tauri Layout

- `src-tauri/src/lib.rs`
  Tauri app entry, Ollama commands, and app setup.

- `src-tauri/src/ai.rs`
  Local AI HTTP server, OpenRouter key handling, and streaming chat runtime.

- `src-tauri/src/ai_data.rs`
  Assistant data tools, store-safe CRUD tooling, and AI-side PowerSync cache integration.

- `src-tauri/tauri.conf.json`
  Tauri app configuration, frontend hooks, window settings, and bundle targets.

- `src-tauri/capabilities/default.json`
  Tauri capability configuration for the main window.

- `src-tauri/gen/android`
  Generated Android project.

- `src-tauri/gen/apple`
  Generated Apple project for iOS packaging.

## PowerSync Layout

- `powersync/schema.sql`
  Source schema for synced tables.

- `powersync/sync-rules.yaml`
  Stream definitions for PowerSync subscriptions.

## Key Build and Config Files

- `package.json`
  Frontend scripts and JS dependencies.

- `vite.config.ts`
  Vite config, TanStack Router plugin, Tailwind plugin, Tauri dev host support.

- `src/env.ts`
  Runtime validation for required `VITE_` environment variables.

- `.env.example`
  Minimal required environment variables.

- `tsconfig.json`
  Strict TypeScript config for `src/`.

## Generated and Derived Files

- `src/routeTree.gen.ts`
  Generated by the TanStack Router plugin. Do not hand-maintain it.

- `dist/`
  Frontend build output.

- `src-tauri/target/`
  Rust build output.

## Where To Look First By Task

- Routing or app shell changes:
  - `src/routes/__root.tsx`
  - `src/routes/index.tsx`
  - `src/routes/stores/$storeSlug.tsx`

- Store feature changes:
  - `src/components/submodules/`

- Auth and organization behavior:
  - `src/auth.ts`
  - `src/lib/organization.ts`

- Data and sync issues:
  - `src/lib/powersync.tsx`
  - `powersync/schema.sql`
  - `powersync/sync-rules.yaml`

- Localization and formatting:
  - `src/lib/i18n.tsx`

- Assistant behavior:
  - `src/components/submodules/assistant-module.tsx`
  - `src/components/submodules/ai-models-module.tsx`
  - `src/lib/assistant.ts`
  - `src/lib/ai-provider.ts`
  - `src-tauri/src/ai.rs`
  - `src-tauri/src/ai_data.rs`
